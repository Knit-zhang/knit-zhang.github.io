docker食用

发表于2022-05-26|更新于2022-05-31|工具
|阅读量:

docker 是PaaS(platform as a server)服务, 与虚拟机不同的是, 虚拟机通过虚拟化技术, 虚拟出硬件, 操作系统等, 而 docker 仅仅虚拟出容器, 来运行服务.
学习一个工具最好的方法就是将它用起来, 将input转为output, 可以帮助我们更好地吸收知识

docker-ce

建议刚接触docker的读者可以先阅读由yeasy撰写的Docker-从入门到实践

docker 的 community edtion(社区版本), 提供docker命令和容器的管理

docker-ce 安装

安装 Docker

检查安装

docker 会自动从 registry pull(拉取)容器, 并运行, 当显示 Hello from Docker:...
时, 表明安装成功

部署常用容器

安装Portainer

Cli 界面虽然更加的灵活, 但是对于新手来说, 图形化界面还是更容易上手, 且在使用portainer的过程中, 可以方便的学会docker的配置方法和步骤

尽管网络上已经有很多程序员做出了 汉化portainer 的image(镜像), 但我还是推荐大家安装官网的版本, 更新也比较快, 唯一的缺点就是没有中文.

建立虚拟卷

拉取容器并安装

第一行是 将本机9443 和8000端口映射到容器内的8000 和 9443 端口, 并将容器命名为portainer
第三行是将本地docker接口挂载
第四行是将portainer_data虚拟卷挂载
第五行是拉取portainer/portainer-ce:latest镜像

访问并初始化

在浏览器上输入 https://<server_ip>:9443/访问portainer

安装 nginx

如果你只有一台云服务器, nginx几乎是你必不可少的工具, 通过nginx你可以实现端口转发(每一个端口都可以拥有一个域名), https(通过下载的ssl证书), 均衡负载等操作, 配置略有些繁琐(最近caddyserver挺流行的,有时间去尝试一下), 但是功能十分强大

将<user_directory>换成你的用户目录, 如/home/kyleten/
将nginx配置文件保存至本地, 既可以方便配置的迁移, 也比较方便配置, 不需要依赖filebrowser等工具来配置 nginx

小结

docker容器化技术十分强大, 有趣的是, docker 的logo是一只背着集装箱的鲸, 也体现了它轻量易迁移的特点, 也减轻了运维的配置压力
